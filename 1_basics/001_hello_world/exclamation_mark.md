
Восклицательный знак `!` в Rust означает, что это **макрос**, а не обычная функция.

## Разница между функцией и макросом

```/dev/null/example.rs#L1-2
println!("Hello");  // это МАКРОС (есть !)
some_function();    // это ФУНКЦИЯ (нет !)
```

## Почему `println!` — это макрос, а не функция?

### 1. Переменное количество аргументов
```/dev/null/example.rs#L1-3
println!("Привет");                          // 1 аргумент
println!("Число: {}", 42);                   // 2 аргумента
println!("{} + {} = {}", 2, 3, 5);           // 4 аргумента
```
Обычные функции в Rust не могут принимать произвольное количество аргументов. Макросы — могут.

### 2. Проверка форматной строки на этапе компиляции
```/dev/null/example.rs#L1-2
println!("{} {}", 1);       // ОШИБКА КОМПИЛЯЦИИ: не хватает аргумента
println!("{}", 1, 2, 3);    // ОШИБКА КОМПИЛЯЦИИ: лишние аргументы
```
Макрос проверяет, что количество `{}` совпадает с количеством аргументов **до запуска программы**.

### 3. Работа с разными типами без обобщений
```/dev/null/example.rs#L1-4
println!("{}", 42);          // i32
println!("{}", 3.14);        // f64
println!("{}", "текст");     // &str
println!("{:?}", vec![1,2]); // Vec<i32>
```

## Другие популярные макросы в Rust

```/dev/null/example.rs#L1-6
println!()      // вывод с переводом строки
print!()        // вывод без перевода строки
format!()       // форматирование в String
vec![]          // создание вектора
panic!()        // аварийное завершение
assert!()       // проверка условия
