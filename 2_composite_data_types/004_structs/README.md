## 📖 Ключевые концепции

### Синтаксис определения структуры

```rust
struct Person {
    name: String,
    age: u8,
    height: f32,
}
```

### Создание экземпляра

```rust
let tom = Person {
    name: String::from("Tom"),
    age: 36,
    height: 1.78,
};
```

### Методы структуры

```rust
impl Person {
    fn new(name: String, age: u8) -> Person {
        Person { name, age, height: 0.0 }
    }
    
    fn greet(&self) {
        println!("Hello, my name is {}", self.name);
    }
}
```

## 💡 Важные моменты

### Ownership и Borrowing
- При передаче структуры в функцию передаётся владение
- Используйте `&` для заимствования без передачи владения
- Используйте `&mut` для изменяемого заимствования

### Copy vs Clone
- **Copy**: автоматическое копирование (только для простых типов)
- **Clone**: явное копирование через `.clone()`
- Структуры со `String` не могут быть `Copy`

### Производные трейты (Derive)

```rust
#[derive(Debug, Clone, PartialEq)]
struct Point {
    x: i32,
    y: i32,
}
```

Доступные трейты:
- `Debug` - форматирование с `{:?}`
- `Clone` - метод `.clone()`
- `Copy` - автоматическое копирование
- `PartialEq` - операторы `==` и `!=`
- `Eq` - полное равенство
- `PartialOrd`, `Ord` - операторы сравнения
- `Hash` - хеширование для HashMap/HashSet

## 🎨 Паттерны проектирования

### Builder Pattern

```rust
let computer = ComputerBuilder::new("Intel i7")
    .ram(16)
    .storage(512)
    .gpu("NVIDIA RTX 3080")
    .build();
```

### Newtype Pattern

```rust
struct Kilometers(f64);
struct Miles(f64);
```

Используется для:
- Типобезопасности
- Реализации трейтов для внешних типов
- Скрытия внутренней реализации

## 📝 Naming Conventions

- **Имена структур**: `PascalCase` (Person, Employee, BookStore)
- **Имена полей**: `snake_case` (first_name, age, is_active)
- **Методы**: `snake_case` (get_name, calculate_age)

## ✅ Best Practices

1. **Инкапсуляция**: делайте поля приватными, предоставляйте публичные методы
2. **Конструкторы**: используйте ассоциированные функции `new()`
3. **Малые структуры**: предпочитайте маленькие, сфокусированные структуры
4. **Builder**: используйте паттерн Builder для структур с множеством полей
5. **Документация**: документируйте публичные структуры и их поля
6. **Ссылки**: передавайте структуры по ссылке (`&`), если не нужна передача владения
